# 执行失败持续学习与主动预防

## 定义

跨领域元 Skill 将可复用的执行失败经验路由到唯一 owner Skill，不建立中央错误大杂烩。

## 操作规则

- 高风险工具调用前进入 prevent，匹配 active 案例后才采用已验证预防动作。
- 非预期失败进入 recover，必须按原输入、原成功标准和 local 环境复验；退出码 0 但产物不可信仍视为失败。
- 复验通过后进入 learn，先脱敏、去重、确认唯一 owner，再写入 candidate。
- 没有当前轮维护授权时不得把 candidate 晋级 active；根因冲突时标记 conflicted 并停止自动复用。
- 业务 Bug 回流 bug-*，Skill 缺口回流 skill-evolution-rules，跨项目稳定知识才进入 Obsidian。
